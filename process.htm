Design Inspiration: Simple but Playful

The design of the website is an attempt to playfully present an interface that is simple in design. A key aspect of web design is to create interfaces that are clean and simple so that they're easily understood by users. Nielsen's 8th heuristic: Aesthetic and Minimalist Design is often quoted for this, emphasising that user interfaces should minimise distractions and keep the focus on the essentials (Nielsen, 1994).

While I always try to keep my designs simple, I wanted to showcase my portfolio in a more creative manner. I decided to make use of bright colours like light green and orange to capture attention, and combine a serif font with a mono font to create contrast. The website also has an overarching motif of rectangular shapes that can be found in the presentation of my projects and global navigation.

Developing the website

This was a simple website developed using vanilla HTML and CSS. Each page is built in a separate HTML page, which is then cross-linked via the navigation bar present at the top of each page. Throughout the site, I attempted to keep my code consistent and easily understandable. To keep the HTML markups organised, I made good use of classes and ids . Every part that would have similar functions and characteristics are given the same class, and ids are further used to differentiate individual elements in a class if necessary.

In terms of CSS, I utilised in-line, internal and external CSS for various purposes. While the bulk of the design relies on the four external CSS stylesheets (main, about, projects and process), I used in-line CSS for one-off styles that would not have to be replicated elsewhere in the site. This way, I could keep the codes cleaner without having to create excess ids to be referenced in an external CSS. I used internal CSS for individual project pages to differentiate between the background colour of the main project page and each individual project.

I also made good use of the flexbox function offered by CSS to easily align items that are presented in groups. For instance, my navigation bar on the top of this document is made up of flex items.

Information Architecture

Information in my website is organised around simplicity and Dan Brown's Principles of Information Architecture in mind. All content are organised with the assumption that the viewer will be a potential employer who would like to learn more about my background and experiences. As such, the information architecture prioritises the "About" and "Projects" pages in the global navigation. I also incorporated many forms of contextual navigation in the form of associative links and a footer to help the viewer find information about me quicker. These are summarised in the following tree diagram.

I kept in mind the following principles (Brown, 2016):
1. Principle of Choices. Offering meaningful choices help viewers find the information they need faster. In my "Projects" page, I include a local navigation that helps a potential employer find the project relevant to the job.

2. Principle of Disclosure. Showing a little information can help viewers understand what they can find when they dig deeper. In my home page, I provided quick links to help a potential employer understand what kinds of projects they can expect to see in this portfolio.

3. Principle of Exemplars. Showing examples help viewers understand the kinds of content on a site. In the "Project Highlights" section, I showcase two examples of projects, selected from the full list of projects in my "Project" page.

Accessibility

As far as possible, I made the portfolio easily usable for individuals with visual impairment and users of screen readers.

Colour Contrast

I treaded carefully in this section as I knew that some of the colours chosen (bright green, orange) are adventurous and may prove to be difficult to read in certain circumstances. To ensure colour contrast, I prioritised the use of dark green as foreground with the bright green as background. While my initial design uses orange a lot more extensively, I toned it down in the current site to ensure colour contrast. In the current site, orange is used as highlights for crucial information.

All foreground-background colour combinations are tested using the WebAIM Contrast Checker to ensure that there is sufficient contrast and enhance readability, with the exception of the use of orange in large texts in certain project pages.


Semantic HTML

As much as possible, I used semantic HTML in my code to aid the users of a screen reader to interpret the website. This includes the following:

  • section
  • article
  • main
  • header
  • footer
  • figure
  • figcaption

Alt Text

All images in this site are given alternative texts, which will display when the image is unavailable.

Reflection and Conclusion

Overall, this has been an enjoyable course and project as it allowed me to recreate my portfolio from scratch. While I've briefly learnt HTML and CSS before, I'm glad that I opted to take this course as it was a good opportunity for me to refresh my knowledge and put it to good practice by building a local website from scratch. I especially appreciated being able to slowly build up the site via the weekly lab activities, as it broke down the complicated process of building a website from scratch into small, actionable steps that were not intimidating to execute.

Besides, I really appreciated that accessibility plays an important part in the course. When I learnt web development in the past, accessible coding was almost never included in the syallbi. I've been able to learn about how to make my source code and designs more accessible for anyone who may require alternative methods of accessing the website. In this website, I've tried my best to make my codes accessible for users with visual impairment, but I acknowledge that there is still much room for improvement, especially with the way that certain sections like buttons and navigation bars are structured.

Moving forward, I do hope to continue refining and expanding this website by introducing responsiveness to different viewport sizes as well as publishing it onto the Internet for easier access. I'd also like to improve the website's accessibility.
